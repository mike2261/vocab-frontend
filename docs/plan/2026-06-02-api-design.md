# API Design

Base URL: `/api`

All protected routes require: `Authorization: Bearer <accessToken>`

## Standard Response Format

**Success:**
```json
{ "data": { ... } }
```

**Success with pagination:**
```json
{
  "data": [...],
  "meta": { "page": 1, "pageSize": 20, "total": 100, "totalPages": 5 }
}
```

**Error:**
```json
{ "error": { "code": "NOT_FOUND", "message": "Vocabulary not found" } }
```

## Auth

### Google OAuth Flow

```
Frontend                        Backend                        Google
   |                               |                              |
   |-- GET /api/auth/google ------>|                              |
   |<-- { url: "https://..." } ----|                              |
   |                               |                              |
   |-- redirect user to url ----------------------->|             |
   |                               |    user grants permission    |
   |<-- redirect to /api/auth/google/callback?code=xxx ----------|
   |                               |                              |
   |          backend exchanges code for Google profile           |
   |          find or create user in DB                           |
   |          issue accessToken + set refreshToken cookie         |
   |                               |                              |
   |<-- redirect to GOOGLE_REDIRECT_FE_URL?token=eyJ... ---------|
```

---

### GET /api/auth/google

No auth required. Initiates Google OAuth.

**Response `200`:**
```json
{ "url": "https://accounts.google.com/o/oauth2/v2/auth?client_id=...&redirect_uri=...&scope=openid email profile&response_type=code" }
```

Frontend redirects the user to this URL.

---

### GET /api/auth/google/callback

Handled by Google redirect. Not called directly by frontend.

Query params set by Google: `?code=xxx&state=xxx`

Backend flow:
1. Exchange `code` for Google access token
2. Fetch Google user profile (`sub`, `email`, `name`, `picture`)
3. Find existing user by `googleId` or `email`; create if new (no `passwordHash`)
4. Issue accessToken + set httpOnly refreshToken cookie
5. Redirect to `GOOGLE_REDIRECT_FE_URL?token=<accessToken>`

**Redirect:** `302` to frontend URL with token in query string.

## Vocabularies

### POST /api/vocabularies

Payload is **word only**. All other fields (meanings, definitions, examples, pronunciation, CEFR level) are generated by LLM.

**Body:**
```json
{ "word": "resilient" }
```

**LLM Generation Flow:**
```
1. Receive { word }
2. Call LLM (OpenRouter) with prompt to generate word data
3. LLM returns structured JSON (meanings, examples, pronunciation, etc.)
4. Save Vocabulary + Meanings + Examples to DB
5. Auto-create ReviewState { stage: 1, intervalDays: 0 }
6. Return full vocabulary object
```

See `.env.example` for `OPENROUTER_API_KEY` and `OPENROUTER_MODEL`.
If `OPENROUTER_API_KEY` is not set, the service returns a mock response.

**LLM prompt template:**
```
Generate a structured dictionary entry for the English word: "{word}"

Return a JSON object with this exact shape:
{
  "pronunciationUk": "IPA string",
  "pronunciationUs": "IPA string",
  "meanings": [
    {
      "partOfSpeech": "noun|verb|adjective|adverb|phrase|...",
      "definition": "English definition",
      "translation": "Vietnamese translation",
      "cefrLevel": "A1|A2|B1|B2|C1|C2",
      "examples": [
        { "sentence": "Example sentence", "translation": "Vietnamese translation" }
      ]
    }
  ]
}

Include 1–3 most common meanings. Include 1–2 examples per meaning.
Return only the JSON object, no markdown, no explanation.
```

**Response `201`:**
```json
{
  "data": {
    "id": "uuid",
    "word": "resilient",
    "pronunciationUk": "/rɪˈzɪl.i.ənt/",
    "pronunciationUs": "/rɪˈzɪl.i.ənt/",
    "note": null,
    "source": "manual",
    "tags": [],
    "meanings": [
      {
        "id": "uuid",
        "partOfSpeech": "adjective",
        "definition": "able to be happy, successful, etc. again after something difficult or bad has happened",
        "translation": "kiên cường, có khả năng phục hồi",
        "cefrLevel": "B2",
        "orderIndex": 0,
        "examples": [
          {
            "id": "uuid",
            "sentence": "She is a resilient person who bounced back after losing her job.",
            "translation": "Cô ấy là người kiên cường và phục hồi nhanh chóng sau khi mất việc.",
            "orderIndex": 0
          }
        ]
      }
    ],
    "reviewState": {
      "stage": 1,
      "intervalDays": 0,
      "repetitionCount": 0,
      "easeFactor": "2.50",
      "nextReviewAt": "2026-06-02T00:00:00.000Z"
    },
    "createdAt": "2026-06-02T00:00:00.000Z"
  }
}
```

### GET /api/vocabularies/:id

Full detail page — word + all meanings + all examples + review state.

**Response `200`:** Same shape as POST /api/vocabularies response `data`.

---

### PATCH /api/vocabularies/:id

Update word-level fields only. To update meanings/examples use their own endpoints.

**Body (all optional):**
```json
{
  "word": "resilient",
  "pronunciationUk": "/rɪˈzɪl.i.ənt/",
  "pronunciationUs": "/rɪˈzɪl.i.ənt/",
  "note": "Useful for interviews",
  "source": "manual",
  "tags": ["interview", "personality"]
}
```

**Response `200`:** Updated vocabulary (same shape as GET /:id).

---

### DELETE /api/vocabularies/:id

Cascades: deletes meanings, examples, reviewState, reviewLogs.

**Response `204`:** No Content

## Reviews

### POST /api/reviews/:vocabularyId

**Body:**
```json
{ "rating": "good" }
```

`rating` must be one of: `forgot` | `hard` | `good` | `easy`

Backend flow (single DB transaction):
1. Load vocabulary + reviewState (check ownership)
2. Run `calculateNextReview(state, rating)` → new intervalDays, easeFactor, repetitionCount, nextReviewAt
3. Run `calculateNextStage(stage, rating)` → new stage
4. Update ReviewState
5. Insert ReviewLog

**Response `200`:**
```json
{
  "data": {
    "vocabularyId": "uuid",
    "stage": 2,
    "repetitionCount": 1,
    "intervalDays": 1,
    "easeFactor": "2.50",
    "lastReviewedAt": "2026-06-02T10:00:00.000Z",
    "nextReviewAt": "2026-06-03T10:00:00.000Z"
  }
}
```

---

## Error Codes

| Code | HTTP | Description |
|------|------|-------------|
| `VALIDATION_ERROR` | 400 | Invalid request body or params |
| `UNAUTHORIZED` | 401 | Missing or invalid access token |
| `FORBIDDEN` | 403 | Authenticated but not the owner |
| `NOT_FOUND` | 404 | Resource not found |
| `CONFLICT` | 409 | e.g. email already registered |
| `LLM_ERROR` | 502 | OpenRouter call failed; mock returned |
| `INTERNAL_ERROR` | 500 | Unexpected server error |
